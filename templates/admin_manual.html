👥1. 사용자 관리

사용자 목록 조회

사이드바 → AI 탭 → 하단 👥 User Management 버튼 클릭

  • 전체 사용자 리스트, 등급, 마지막 로그인 시각 확인
  • 활성/비활성 상태 관리

사용자 추가

  1. User Management → + 사용자 추가 버튼
  2. 아이디, 표시 이름, 등급 입력
  3. 초기 비밀번호 설정 (6자 이상)
  4. 생성 클릭 → 사용자에게 아이디/비밀번호 전달

비밀번호 변경 (관리자)

사용자 목록에서 해당 유저의 🔐 비밀번호 변경 버튼 클릭:

  • 새 비밀번호 입력 (6자 이상)
  • 🎲 랜덤 버튼으로 자동 생성 가능
  • 변경 후 비밀번호가 화면에 표시됨 → 📋 복사 버튼으로 복사
비밀번호는 SHA-256 + Salt로 해시되어 저장됩니다. 평문은 DB에 저장되지 않습니다.

사용자 목록에서 등급 드롭다운 변경 후 저장:

등급권한

Director대부분 기능, 11개 모델 접근

사용자 비활성화 / 삭제

  • 비활성화: 토글 스위치 OFF → 로그인 차단 (데이터 보존)
  • 삭제: 🗑️ 버튼 → 확인 팝업 → 영구 삭제
⚠️ 삭제된 사용자의 대화 기록, 워크스페이스 파일은 복구 불가합니다.

모델 추가 방법

  1. app.py → MODEL_CATALOG 리스트에 새 항목 추가
  2. 형식: {"id": "provider:model", "provider": "...", "model": "...", "label": "...", "cost": "low|medium|high"}
  3. TIER_MODELS 딕셔너리에서 접근 허용할 등급에 모델 ID 추가
  4. auto_route_model() 함수에서 라우팅 로직 업데이트
  5. 커밋 & 푸시 → Render 자동 배포
💡 /update-models 워크플로우를 사용하면 AI 어시스턴트가 자동으로 최신 모델을 검색하고 업데이트합니다.

🗄️7. 데이터베이스 (Supabase)

주요 테이블

테이블용도주요 컬럼
users사용자 계정id, username, password_hash, tier, display_name, is_active
conversations대화 기록id, user_id, title, messages (JSON), created_at
workspace_folders워크스페이스 폴더id, user_id, name, icon, description
workspace_files워크스페이스 파일id, folder_id, title, content, file_type
memories장기 기억id, user_id, content, category
personasAI 페르소나id, user_id, name, job, personality

Supabase 관리

  • 대시보드: supabase.com → 프로젝트 → Table Editor
  • RLS (Row Level Security): users 테이블에 RLS 활성화됨 → supabase_admin (service key)으로만 관리 가능
  • 백업: Supabase Pro 플랜에서 자동 백업 지원
⚠️ supabase_client(anon key)로는 users 테이블 UPDATE가 불가합니다. 반드시 supabase_admin 사용하세요.

🚀8. 배포 (Render)

배포 프로세스

  1. 코드 변경 → git push → GitHub에 푸시
  2. Render에서 자동 감지 → 빌드 & 배포 시작
  3. 빌드: pip install -r requirements.txt
  4. 시작: gunicorn app:app (Procfile 참조)
  5. 배포 완료 → firstaibot.com에서 확인

Render 설정

  • 서비스 타입: Web Service
  • 브랜치: master
  • 빌드 커맨드: pip install -r requirements.txt
  • 시작 커맨드: gunicorn app:app
  • Auto-Deploy: ON (git push 시 자동 배포)
  • 커스텀 도메인: firstaibot.com

🎭9. 페르소나 & 템플릿 관리

기본 페르소나 (75개+)

파일: app.py → PERSONAS 딕셔너리

  • 각 페르소나는 이름, 직업, 성격, 전문 분야, 응답 스타일 정의
  • 사용자가 커스텀 페르소나 생성 가능 (DB에 저장)
  • President는 모든 페르소나 관리 가능

사용자 커스텀 템플릿

API: /api/user-templates

  • 사용자가 자주 사용하는 문서 템플릿을 DB에 저장
  • 템플릿 이름, 설명, 프롬프트로 구성

📂10. 워크스페이스 & 메모리

워크스페이스

  • 사용자별 클라우드 폴더/파일 시스템
  • 대화, 보고서, 슬라이드를 폴더에 저장
  • 파일 업로드 (PDF, Excel 등) 지원
  • 파일 핀 고정, AI 연계 기능 (Ask AI / Continue / Develop)

장기 기억 시스템

  • AI가 대화에서 핵심 사실을 자동 추출하여 저장
  • 이후 대화에서 자동으로 참조
  • 사용자가 수동 추가/삭제 가능
  • Guest 계정은 메모리 비활성

📊11. 모니터링 & 로그

Render 로그

  • Render Dashboard → 서비스 → Logs 탭
  • 실시간 서버 로그 확인 (요청, 에러, API 호출)
  • 각 AI 요청에 대한 모델, 응답 시간, 토큰 수 로깅

주요 모니터링 포인트

  • API 키 만료: 각 프로바이더 대시보드에서 사용량/만료일 확인
  • Supabase 사용량: Free 플랜 500MB DB, 2GB 전송 제한
  • Render 사용량: Free 플랜 750시간/월, 자동 슬립(15분 비활성)
  • 에러율: 429 (Rate Limit), 500 (서버 에러) 빈도 확인

🛡️12. 보안 가이드

인증 보안

  • 비밀번호 해시: SHA-256 + Salt (PASSWORD_SALT 환경변수)
  • 세션 관리: Flask 서버 사이드 세션, SECRET_KEY로 암호화
  • Rate Limiting: IP 기반 요청 제한으로 브루트 포스 방지
  • 로그인 시도 제한: 과다 로그인 시도 차단

보안 체크리스트

  • ☐ 모든 API 키를 환경 변수로 관리 (코드에 미포함)
  • ☐ SECRET_KEY를 강력한 랜덤 값으로 설정
  • ☐ PASSWORD_SALT를 고유 값으로 설정
  • ☐ Guest 계정 비밀번호를 주기적으로 변경
  • ☐ Supabase RLS 정책 확인
  • ☐ HTTPS 적용 확인 (Render 자동 제공)

🔧13. 트러블슈팅

증상원인해결
로그인 실패 (Invalid credentials)비밀번호 해시 불일치관리자에서 비밀번호 재설정. supabase_admin 사용 확인
AI 응답 없음API 키 만료/잘못됨Render 환경변수에서 API 키 확인
429 Too Many RequestsRate Limit 초과등급 업그레이드 또는 1분 대기
502/503 에러서버 슬립 또는 빌드 실패Render 대시보드에서 로그 확인, 수동 배포
대화 저장 안 됨Supabase 연결 문제SUPABASE_URL, SUPABASE_KEY 확인
웹 검색 안 됨SerpAPI 키 누락SERPAPI_KEY 환경변수 설정
이미지 생성 안 됨DALL-E API 권한 없음OpenAI 계정에서 DALL-E 접근 확인
음성 입력 안 됨브라우저 마이크 권한Chrome/Edge에서 마이크 권한 허용

🔄14. 정기 유지보수

월간 체크리스트

  • ☐ AI 모델 업데이트: /update-models 워크플로우 실행
  • ☐ API 키 확인: 각 프로바이더 대시보드에서 사용량/잔액 확인
  • ☐ Supabase 사용량: DB 크기, 전송량 확인
  • ☐ Render 사용량: 빌드 시간, 대역폭 확인
  • ☐ Guest 비밀번호: 필요시 변경
  • ☐ 비활성 사용자: 장기 미사용 계정 정리

비용 최적화 팁

  • Auto 라우팅 활용: 간단한 질문에 고비용 모델 사용 방지
  • 등급 적절 배정: 불필요하게 높은 등급 할당 금지
  • DeepSeek 활용: 중국어/일반 질문에 저비용 DeepSeek 모델 사용
  • Gemini Flash: 빠른 응답이 필요한 Staff/Guest에 Gemini Flash 배정
